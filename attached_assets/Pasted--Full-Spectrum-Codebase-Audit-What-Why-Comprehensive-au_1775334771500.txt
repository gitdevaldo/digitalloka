# Full-Spectrum Codebase Audit

## What & Why
Comprehensive audit of the DigitalLoka codebase covering security, database integrity, performance, code quality, UX, and future risks. The goal is to fix all autonomously fixable issues and document anything requiring human decision.

## Done looks like
- All fixable issues listed below are resolved in the codebase
- Each fix is documented with area, severity, finding, impact, and action taken
- Issues requiring human decision are flagged with reasoning
- A summary audit report is generated at `docs/audits/2026-04-04-full-spectrum-audit.md`

## Out of scope
- Destructive schema migrations (flagged for human decision)
- Business logic changes (e.g., changing pricing model)
- UI redesign — only fixing broken/missing states
- Adding new features

## Tasks

### Critical & High Severity Fixes

1. **Webhook signature bypass hardening** — The Mayar webhook at `src/app/api/payments/mayar/webhook/route.ts` (lines 37-46) skips signature verification when `MAYAR_SANDBOX=true`. The guard only blocks `isSandbox && isProduction`, meaning non-production environments with real data are unprotected. Harden this: always verify signatures unless explicitly in local dev, and log a warning if sandbox mode is active.

2. **Missing error checking in auth service** — `isAdmin()` and `getUserDropletIds()` in `src/lib/services/supabase-auth.ts` destructure `data` from `.single()` calls without checking the `error` object. If the query fails (connection issue, RLS error), these silently return `false` / `[]` — potentially locking admins out or hiding droplets. Add explicit error handling and logging.

3. **Use `.maybeSingle()` instead of `.single()` for lookups** — Multiple locations use `.single()` where "not found" is a valid case: category lookups in `src/app/api/admin/products/route.ts` (lines 50, 54, 60, 64) and `src/app/api/admin/products/[id]/route.ts`, user lookups in `src/app/auth/callback/route.ts` and `src/app/api/auth/session/route.ts`. Replace with `.maybeSingle()` to avoid unnecessary 406/PGRST116 errors.

4. **Admin API missing pagination** — `src/app/api/admin/users/route.ts` and `src/app/api/admin/products/route.ts` fetch ALL records with `.select('*')` and no `limit`/`range`. This will degrade as data grows. Add server-side pagination with offset/limit query params.

### Medium Severity Fixes

5. **Deduplicate category resolution logic** — Category ID/name resolution code is duplicated between `src/app/api/admin/products/route.ts` (lines 36-67) and `src/app/api/admin/products/[id]/route.ts` (lines 39-72). Extract into a shared helper in the services layer.

6. **Inconsistent API response format** — `src/app/api/user/cart-checkout/route.ts` uses raw `NextResponse.json` instead of `apiSuccess`/`apiError` helpers. `src/app/api/admin/users/route.ts` also uses raw `NextResponse.json`. Standardize all routes to use the response helpers from `src/lib/api-response.ts`.

7. **Stock counting pulls all rows into memory** — `src/app/api/products/route.ts` (lines 31-46) fetches every `product_stock_items` row matching the product IDs just to count them. Replace with a database-level `COUNT` + `GROUP BY` query or an RPC to avoid loading potentially thousands of rows.

8. **Missing loading states in admin UI** — `src/app/admin/(dashboard)/product-stocks/[id]/edit/page.tsx` uses `if (loading) return null` instead of showing a skeleton or spinner. Add proper loading indicators.

9. **Price calculation logic scattered** — Price logic exists in both `src/app/(public)/cart/page.tsx` (local `getItemPrice`) and `src/app/api/user/cart-checkout/route.ts`. Centralize into a shared utility to prevent mismatches.

10. **Dynamic imports for heavy libraries** — `xlsx` and `react-quill-new` are statically imported, bloating the initial bundle. Use `next/dynamic` or dynamic `import()` so they only load when needed (export button click, editor mount).

### Low Severity Fixes

11. **Fire-and-forget audit logging** — Audit log calls use `.catch()` to silently swallow failures. Add structured logging so audit failures are trackable in monitoring, even if they don't block the response.

12. **Unused imports and dead code cleanup** — Remove unused imports across API routes and clean up utility scripts (`scripts/post-merge.sh`, `scripts/create-provider-data.mjs`) that aren't wired into the project.

13. **DigitalOcean cache fallback noise** — `src/lib/services/digitalocean.ts` logs to console on every cache miss fallback, which can spam logs. Add log-level gating or debouncing.

14. **Docs folder exposure** — Ensure `docs/` directory (containing architecture details, audit reports, policy matrices) is excluded from production builds via `next.config.js` or `.dockerignore` equivalent.

### Audit Report

15. **Generate audit report** — Write a summary document at `docs/audits/2026-04-04-full-spectrum-audit.md` listing each finding with area, severity, file path, impact, and action taken (fixed vs. flagged for human decision).
